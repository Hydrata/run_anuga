# TASK-2156 (epic 2147 W2) — mass-balance e2e RED-PROOF

`tests/test_mass_balance.py` builds a small, flat, fully-Reflective (closed
— no mass leaves) synthetic domain via `anuga.rectangular_cross_domain`,
applies a known uniform hyetograph (and, in the "aligned" variant, a
constant Surface inflow) via the REAL `apply_inflows_to_domain` +
`anuga.Polygonal_rate_operator` / `anuga.Inlet_operator`, evolves the REAL
domain, and asserts `domain.get_water_volume()` matches the independently
computed expected volume within 2%.

The expected-volume math deliberately hardcodes its own conversion
constant (`_EXPECTED_RAINFALL_FACTOR = 1.0 / (1000.0 * 3600.0)`) instead of
importing `run_anuga.defaults.RAINFALL_FACTOR` — otherwise a regression
that corrupts the production constant would move both sides of the
assertion together and the test would stay green while blind to exactly
the failure class it exists to catch.

## RED-PROOF transcript (manual, run once during delivery)

Real `RAINFALL_FACTOR = 1.0 / (1000.0 * 3600.0)` ≈ 2.7778e-7. Temporarily
set it to `1e-6` (~3.6x larger) in `run_anuga/defaults.py`:

```
$ sed -i 's/^RAINFALL_FACTOR = 1.0 \/ (1000.0 \* 3600.0).*/RAINFALL_FACTOR = 1e-6  # TEMP RED-PROOF/' run_anuga/defaults.py
$ /opt/venv/anuga-core-dev/bin/python -m pytest -k mass_balance -q
```

Result — **RED**, both variants fail, off by exactly the expected ~3.6x:

```

test_mass_balance_aligned_constant_inflow_plus_rain
```

a wrong `RAINFALL_FACTOR` (this transcript) and a broken model_start/
timestamp alignment (a total mismatch either raises via the TASK-2155
NaN guard or silently applies zero rain — both fail this test's volume
assertion loudly, by construction of the assertion itself).
